# Training Data Guide — Using 2000+ Lesson JSONs

This file specifies how to build and use the training data pipeline
for AI-powered lesson generation in Animo Studio.

---

## Overview

The `reference/json-lessons/` folder is a growing library of real lesson JSON files
from the BrightChamps platform. Starting with 10 files, growing to 2000+.

These JSONs are **not used at runtime by students**. They are:
1. Training examples for the AI lesson generator
2. Pattern validation data for the lesson quality checker
3. Template scaffolding source for the `/new-lesson` command

---

## Folder Conventions

```
reference/json-lessons/
  data_8.json              ← Original batch (moduleId: FINX101)
  data_9.json              ← (moduleId: FINX102)
  data_10.json             ← (moduleId: FINX103)
  ...
  data_17.json             ← (moduleId: YTL10301)
  patterns-index.json      ← AUTO-GENERATED by analyzer (do not edit manually)
  metadata.json            ← AUTO-GENERATED summary stats
```

When adding new JSONs, any filename format works. The analyzer reads all `.json` files
except `patterns-index.json` and `metadata.json`.

## Pattern Analyzer (`/lib/trainingData/patternAnalyzer.ts`)

### What it does

Reads all lesson JSONs and produces a `patterns-index.json` with:

```typescript
interface PatternsIndex {
  generatedAt: string               // ISO timestamp
  totalModules: number
  totalTasks: number
  templateFrequency: Record<string, number>
  arcPatterns: ArcPattern[]         // extracted sequence fingerprints
  dialogueLengths: {
    opening: { min: number, max: number, avg: number }
    midLesson: { min: number, max: number, avg: number }
    reaction: { min: number, max: number, avg: number }
  }
  subjectProfiles: Record<string, SubjectProfile>
  qualityRules: QualityRule[]       // validated from real data
}

interface ArcPattern {
  id: string
  name: string                      // e.g. "Standard 30-task financial module"
  moduleIds: string[]               // which modules match this pattern
  templateSequence: string[]        // abbreviated e.g. ["C","C","R","C","M","C","R","M","M","F"]
  // C=conversation, R=readonly, M=mcq, F=feedback, etc.
  taskCount: { min: number, max: number }
  subject: string
  gradeLevel: string
}

interface SubjectProfile {
  dominantTemplates: string[]
  averageTaskCount: number
  activityTypes: string[]
  characterCount: { min: number, max: number, avg: number }
}

interface QualityRule {
  id: string
  description: string
  check: string                     // code expression as string
  severity: 'error' | 'warning'
}
```

### How to run

```bash
# From project root:
npx ts-node lib/trainingData/patternAnalyzer.ts

# Or via Claude Code command:
/analyze-training-data
```

### Implementation sketch

```typescript
// lib/trainingData/patternAnalyzer.ts

import fs from 'fs'
import path from 'path'

const LESSONS_DIR = path.join(process.cwd(), 'reference/json-lessons')

function analyzeAll() {
  const files = fs.readdirSync(LESSONS_DIR)
    .filter(f => f.endsWith('.json') && f !== 'patterns-index.json' && f !== 'metadata.json')

  const modules = files.map(f => ({
    filename: f,
    tasks: JSON.parse(fs.readFileSync(path.join(LESSONS_DIR, f), 'utf-8'))
  }))

  const index: PatternsIndex = {
    generatedAt: new Date().toISOString(),
    totalModules: modules.length,
    totalTasks: modules.reduce((sum, m) => sum + m.tasks.length, 0),
    templateFrequency: countTemplates(modules),
    arcPatterns: extractArcPatterns(modules),
    dialogueLengths: measureDialogueLengths(modules),
    subjectProfiles: buildSubjectProfiles(modules),
    qualityRules: DERIVED_QUALITY_RULES,
  }

  fs.writeFileSync(
    path.join(LESSONS_DIR, 'patterns-index.json'),
    JSON.stringify(index, null, 2)
  )

  console.log(`Analyzed ${index.totalModules} modules, ${index.totalTasks} tasks.`)
}
```

## Lesson Generator (`/lib/trainingData/lessonGenerator.ts`)

### Phase 1: Rule-based (current)

Generates lesson from topic using the Golden Arc + subject profile:

```typescript
interface GenerationInput {
  topic: string
  subject: SubjectArea
  gradeLevel: GradeLevel
  lessonLength: 'short' | 'medium' | 'long'
  characterName: string
}

function generateLesson(input: GenerationInput): Scene[] {
  // 1. Select arc based on lessonLength
  const arc = lessonLength === 'short' ? SHORT_ARC : STANDARD_ARC

  // 2. Apply subject profile (e.g. financial → more MCQ, tycoons → more tables)
  const profile = SUBJECT_PROFILES[input.subject]

  // 3. Scaffold scenes from arc template
  return arc.map((arcStep, i) => ({
    id: generateId(),
    order: i + 1,
    script: SCRIPT_TEMPLATES[arcStep.tag].replace('[TOPIC]', input.topic),
    tag: arcStep.tag,
    animationState: SCENE_TAG_ANIMATIONS[arcStep.tag],
    background: arcStep.background || 'white',
    approved: false,
    flagged: false,
    comments: [],
  }))
}
```

async function generateLessonWithAI(input: GenerationInput): Promise<Scene[]> {
  // 1. Load patterns-index.json

  // 2. Find closest matching arc pattern

## Quality Validator (`/lib/trainingData/qualityValidator.ts`)

Uses rules derived from training data to validate teacher-created lessons:

```typescript
const QUALITY_RULES: QualityRule[] = [
  {
    id: 'Q001',
    description: 'Never more than 6 consecutive conversation scenes without an activity',
    severity: 'warning',
    check: (scenes: Scene[]) => {
      let consecutive = 0
      for (const scene of scenes) {
        if (scene.tag === 'explain' || scene.tag === 'think') consecutive++
        else consecutive = 0
        if (consecutive > 6) return false
      }
      return true
    }
  },
  {
    id: 'Q002',
    description: 'Lesson must end with a celebration/feedback scene',
    severity: 'error',
    check: (scenes: Scene[]) => {
      const last = scenes[scenes.length - 1]
      return last?.tag === 'celebrate'
    }
  },
  {
    id: 'Q003',
    description: 'At least one activity (non-conversation, non-readonly) required',
    severity: 'error',
    check: (scenes: Scene[]) => {
      return scenes.some(s => s.tag === 'question' || s.tag === 'think')
    }
  },
  {
    id: 'Q004',
    description: 'Minimum 3 scenes recommended for meaningful lesson',
    severity: 'warning',
    check: (scenes: Scene[]) => scenes.length >= 3
  },
  {
    id: 'Q005',
    description: 'Script text should be 10-100 words per scene',
    severity: 'warning',
    check: (scenes: Scene[]) => {
      return scenes.every(s => {
        const words = s.script.split(' ').length
        return words >= 5 && words <= 120
      })
    }
  },
]
```

Show quality warnings in the ReviewPanel before allowing status advance.
Q001 and Q004 = warnings (can override). Q002 and Q003 = errors (must fix).

## How to Add 2000+ Lesson JSONs

1. Export lesson JSONs from the BrightChamps CMS (or any source)
2. Drop files into `reference/json-lessons/` — any filename works
3. Run `/analyze-training-data` in Claude Code to re-index
4. Check `reference/json-lessons/metadata.json` for updated stats
5. If new template types appear, add them to `TASK_TEMPLATE_CATALOG.md`

**No code changes needed for additional JSONs.** The pattern analyzer and lesson
generator read the index dynamically at runtime.

## Privacy & Data Notes

- Lesson JSONs contain only content/structure data, no student PII
- Character names (Greenline, Max) are platform assets, not real people
- Asset URLs in JSONs (`/assets/...`) are internal CDN paths — they will 404
  in the Animo Studio dev environment, which is expected and OK
- Never commit student submission data or user IDs to this folder
